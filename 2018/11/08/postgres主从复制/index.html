postgresql主从复制

发表于 2018-11-08 | 分类于 Study
postgreSQL + postGis 的主从复制搭建。

官方传送门

https://www.postgresql.org/docs/10/different-replication-solutions.html

环境说明

服务器系统：ubuntu 16.04.04 LTS

数据库版本：postgreSQL 10 + postGIS 2.4

主服务器IP：192.168.0.1

从服务器IP：192.168.0.2

更新APT

执行：
sh -c ‘echo “deb http://apt.postgresql.org/pub/repos/apt xenial-pgdg main” >> /etc/apt/sources.list’

执行：
wget –quiet -O - http://apt.postgresql.org/pub/repos/apt/ACCC4CF8.asc | sudo apt-key add -

执行：
apt update

安装数据库主程序

执行：
apt install postgresql-10

执行：
apt install postgresql-10-postgis-2.4

执行：
apt install postgresql-10-postgis-scripts

执行：
apt install postgis

执行：
apt install postgresql-10-pgrouting

设置数据库

https://trac.osgeo.org/postgis/wiki/UsersWikiPostGIS24UbuntuPGSQL10Apt

切换用户：

su    postgres

登陆控制台

psql

执行：

CREATE EXTENSION adminpack;

执行：

CREATE DATABASE gisdb;

执行：

\connect gisdb;

执行：

CREATE SCHEMA postgis;

执行：

ALTER DATABASE gisdb SET search_path=public, postgis, contrib;

执行：

\connect gisdb;

配置空间拓展，执行：

CREATE EXTENSION postgis SCHEMA postgis;
SELECT postgis_full_version();

配置成功会显示postgis_full_version相关信息。

配置3D拓展，执行：

CREATE EXTENSION postgis_sfcgal SCHEMA postgis;
SELECT postgis_full_version();

配置成功会显示新的postgis_full_version相关信息。

执行：

CREATE  EXTENSION pgrouting SCHEMA postgis;
SELECT * FROM pgr_version();

配置成功会显示版本信息。

修改pg_hba.conf配置文件，将：

local         all           all        peer

修改为

local         all           all        trust

文件目录：/etc/postgresql/10/main/pg_hba.conf

重启服务器

执行：

service postgresql restart

配置主服务器

创建同步用户
su postgres

创建用户：用于同步数据

执行：

create role replimloge login replication encrypted password 'replication'

命令说明：replimloge，用户名；login、replication，权限类型

配置pg_hba.conf

修改pg_hba.conf，添加如下

host    all          all           192.168.0.2/32   trust   #从服务器连接到本机

host    replication  replimloge    192.168.0.2/32    md5    #replimloge用户在从服务器上复制本机数据，角色必须是 replication
配置postgresql.conf

文件位置：/etc/postgresql/10/main/postgresql.conf

执行：

vim /etc/postgresql/10/main/postgresql.conf

修改如下信息：

listen_addresses = '*'                                     #监听所有ip

#archive_mode = on                                          #开启归档模式

archive_command = ‘cp %p /var/lib/postgresql/10/main/%f’   #归档命令

wal_level = replica                                        #热备模式

max_wal_senders = 10                                       #最多有10个流复制连接

wal_keep_segments = 100                                    #最多有多少个同步日志文件

wal_sender_timeout = 60s                                   #流复制超时时间

说明：archive_mode = on 会打开归档模式，生成大量wal日志文件。wal_keep_segments 根据具体需求设定，如果设置的过小，大量数据写入时可能会导致同步文件被覆盖，而导致同步异常。

重启数据库

执行：

service postgresql restart

配置从服务器

测试与主服务器的连接

执行：

psql -h 192.168.0.1  -U  postgres
清空数据目录

执行：

rm -rf /var/lib/postgresql/10/main/*
同步备份主服务器数据

执行：

pg_basebackup -F p --progress -D 10/main/ -h 192.168.0.1 -p 5432 -U replimloge --password
创建recovery.conf

recovery.conf文件，用于从库恢复从主库获取的数据。

执行：

vi recovery.conf

写入如下信息：

standby_mode = on   #表示该节点是从库

primary_conninfo = 'host=192.168.0.1  port=5432  user=replimloge  password=mlogcn'  #从机信息和连接用户

保存退出

移动到数据目录：10/main/.

执行：

mv recovery.conf 10/main/.
配置postgresql.conf

执行：

vim /etc/postgresql/10/main/postgresql.conf

修改如下信息：

listen_addresses = '*' 

wal_level = replica                 #热备模式

hot_standby = on                    #说明这台机器不仅用于数据归档，还可以用于数据查询

max_standby_streaming_delay = 30s   #流备份的最大延迟时间

wal_receiver_status_interval = 10s  #向主机汇报本机状态的间隔时间

hot_standby_feedback = on           #出现错误复制，向主机反馈

保存退出，重启数据库

执行：

service postgresql restart

测试

在主服务器中创建一个表，并写入数据，看从服务器是否正确同步。

综上，上述就是主从同步配置的全过程。
